# Design Audit Report

> Generated by 3 parallel audit sub-agents (visual via Playwright MCP, code-level static, cross-component consistency) — Friday, May 1, 2026.

The application — `Reimbursement Request Manager` for Novi Community School District — was just rebuilt on shadcn/ui defaults (slate base color, 0.5rem radius, default font stack). The functional rewrite is complete (all tests green). The user asked for a design pass to surface inconsistencies, ugliness, distasteful color, and bad UI.

This report consolidates the three sub-reports:

- [`visual-audit.md`](visual-audit.md) — page-by-page visual findings with 38 screenshots
- [`code-audit.md`](code-audit.md) — source-level style audit
- [`consistency-audit.md`](consistency-audit.md) — cross-component drift audit

---

## TL;DR — top 5 issues

1. **The brand promise is broken.** The Novi logo is bright emerald green; the rest of the UI is uniform slate-on-white with zero accent color. The product looks like the shadcn template with a green logo PNG strapped on. *(Visual: every screenshot. Code: `globals.css` slate OKLCH variables; no `next/font` for a brand font.)*
2. **Typography is an accident.** No `next/font` is loaded in `src/app/layout.tsx`. `--font-heading` is aliased to `--font-sans` (`globals.css:7-8`). PageHeader `<h1>` renders in the serif system fallback (looks like Times). `/policy` body text falls into the same accidental serif cascade. There is no display face anywhere — and there *should* be.
3. **The Admin Inbox doesn't match its name.** 7 of 8 cards visible on `/admin/inbox` carry an `Admin Approved` badge. Either the seed misleads or the query filter is broken. Either way, an admin lands on "Inbox" and sees mostly already-approved work.
4. **The editable request header has no editable affordance.** On a draft request detail, title and description render as plain text with no border, no background, no hover cue. Users have no way to know they can click to edit.
5. **`/coach/inbox` doesn't exist.** The route silently redirects to `/coach/team-reimbursements`. The navbar entry promises a separate inbox UI. There isn't one.

### MINOR

- **Two greens in play** — Novi logo green ≠ status-badge emerald.
- **PDF download** on detail pages is tiny grey "↓ PDF" with no button chrome — easy to miss.
- **Manage Teams** dominated by `--` placeholders and `0` columns.
- **Stat tiles are flat** — no icons, trends, or color cues.
- **Notification popover** items have no icons, no read/unread state, no day separators.
- **Sign-in card** sits alone on a vast white desert — no atmosphere, no Novi imagery beyond the logo.
- **Sales tax strikethrough** is clever but reads as "this number is wrong."
- **Parent-mentor dashboard** has only 2 tiles in a 3-column lg grid — empty third column.
- **PageHeader nesting** in auth pages uses wrapper hacks (`[&_[data-slot=card-header]]:...`) that don't exist on app pages.
- **`profile-form.tsx`** uses `space-y-6` while every other form uses `space-y-4`.
- **`request-form.tsx`** swaps button label inline (`"Creating..."`) instead of using the shared `Button` `loading` prop.
- **Dialog title tone drift** — "Delete this receipt?" (question) vs "Remove member" (statement) vs "Create team" (statement).
- **Section heading scale heterogeneous** — `<h2>` is `text-sm` here, `text-xl` there, `text-lg` elsewhere.
- **Card-like blocks bypass the Card primitive** — `profile-form.tsx`/`sign-up-form.tsx` use `rounded-md border ... bg-muted/50` callouts instead of nested Cards.
- **Loading-skeleton density varies** by route's `cardCount` / `lines` props.
- **Icon size drift** — `size-4`/`size-5` mostly, but `size-6` for hero icons and `size-8` for upload zone.
- **Navbar signed-out CTAs** reimplement Button styling inline (`navbar.tsx:67-71`) rather than composing `<Button asChild>`.

### NIT

- No avatar / initials anywhere — user identity is raw email.
- No background texture, gradient, illustration, or decorative element on any page.
- No section dividers beyond thin slate borders.
- Italics used only as "empty placeholder" affordance ("No coach assigned") — never for emphasis.
- `tabular-nums` used in exactly one place (`line-item-comments.tsx:173`).

## What works

To balance the negative findings:

- **Functional consistency is strong**: forms use rhf+zod+`<Form>` everywhere they should; tables use `DataTable`+tanstack; toasts via Sonner; destructive flows via `AlertDialog`; navigation primitives (`NavigationMenu`, `Sheet`, `Popover`) used where appropriate. *(See verification report from prior session.)*
- **Profile page** is the cleanest screen in the app — clear card → form → save flow.
- **Status timeline** on detail pages reads cleanly.
- **Sortable column headers** are unified across all 7 tables (after the recent `SortableColumnHeader` consolidation).
- **PageHeader** is used on every real-content page (the wrapper hack on auth pages aside).
- **EmptyState** is used on every list-empty case (with the noted gaps in filtered/popover empty states).
- **Form validation** works correctly — `screenshots/interactions/reject-no-comment.png` proves "Rejection requires a comment." appears via `FormMessage`.
- **Mobile nav** collapses to hamburger + `Sheet` correctly.
- **0 inline SVGs**, **0 stray Tailwind palette utilities** outside documented exceptions, **0 legacy primitive imports** (verified in prior session).

## Recommendations (ranked)

### Tier 1 — fix the typography pipeline (1 day)

1. Load a real type system in `src/app/layout.tsx` via `next/font`:
   - **Body sans**: e.g. `Geist Sans` (ironic but actually load it), `Inter`, or for Novi character: `Public Sans` / `Plus Jakarta Sans` / `IBM Plex Sans`.
   - **Display heading**: a distinctive face — `Fraunces`, `DM Serif Display`, `Manrope` (semi-display), `Bricolage Grotesque`, or `Geist Mono` for a developer-tool feel. Pick *one* and own it.
   - **Mono** for receipt totals and IDs: `JetBrains Mono` or `Geist Mono`.
2. Replace the `--font-heading: var(--font-sans)` alias in `globals.css` with `--font-heading: var(--font-display)`.
3. Verify `/policy` and all `PageHeader` h1s render in the chosen display face — not the system serif fallback.
4. Bump the type scale: PageHeader `<h1>` from `text-2xl` to `text-3xl sm:text-4xl`. Detail page money totals (`text-2xl`) → `text-4xl tabular-nums`.

### Tier 2 — adopt a brand color system (1 day)

5. Add a Novi-green primary token:
   - Sample the logo green (likely `#1F7A3E` or similar).
   - Set `--primary: <oklch>` in `globals.css :root` and `.dark`.
   - Optional: add `--primary-50` through `--primary-900` for subtle backgrounds, hovers, focus rings.
6. Re-skin status badges to use semantic chart-* tokens or a curated 3-color palette derived from primary, not rainbow Tailwind tints.
7. Apply the brand color: primary CTAs, focus rings, hover accents on dashboard tiles, sortable column header active state, sign-in CTA, empty-state icon tints.

### Tier 3 — fix the page-level visual issues (1-2 days)

8. **Hide the empty "Receipts & line items" card** on admin detail when no content, OR render `<EmptyState>`. Same for any other section that conditionally has content.
9. **Fix `/admin/inbox` query** so it only shows requests in `SUBMITTED` / `COACH_APPROVED` states — not `Admin Approved`.
10. **Add visible affordances to `EditableRequestHeader`**: input borders, "click to edit" hover state, or convert to inline-edit pattern with explicit Edit button.
11. **Restyle Reject button**: filled red `bg-destructive text-destructive-foreground` instead of the muted variant. Reserve the muted variant for less-final actions.
12. **Decide on `/coach/inbox`**: either build it (filtered queue of items needing coach review) or remove the navbar entry.
13. **Fix mobile draft detail line items**: collapse to a card list (one card per line item) below 768px instead of a cramped 5-col table. Or use a horizontal-scroll table with a sticky description column.

### Tier 4 — consistency & polish (1 day)

14. **Centralize formatting**: one `formatCurrency()` (always `$X.XX`), one `formatDate()` (e.g. `MMM D, YYYY`), one `formatRelativeTime()`. Replace all ad-hoc usage.
15. **Unify failure surfaces**: bring `global-error.tsx` and `forbidden.tsx` to the same Card pattern as `error.tsx`/`not-found.tsx`/`unauthorized.tsx`. Use one CTA vocabulary ("Go home" / "Try again").
16. **Unify empty states**: extend `<EmptyState>` to handle filtered-empty in tables and notification-empty cases.
17. **Stat tiles**: add an icon, a subtle background hue, and a delta indicator where relevant. Stop showing `0` columns; suppress the column or show "—" muted.
18. **Status badge casing**: pick one rule (Title Case for everything) and remove `LABEL_OVERRIDES` for legacy ALL CAPS.
19. **Remove dev-tool overlay button** on production builds (`next.config.ts`).

### Tier 5 — distinction (optional, 2-3 days)

20. **Backgrounds with character**: a subtle dot grid on the auth shell (`bg-[radial-gradient(...)]`), a soft gradient at the top of the dashboard, a Novi pattern in empty states.
21. **Microinteractions**: tile hover lift (`hover:-translate-y-0.5 hover:shadow-md transition-all`), card focus ring on tab, success-toast slide-and-bounce.
22. **Avatar component**: derive initials from email/name; render in tables and the navbar.
23. **Notification list**: group by day, add `Mark all read`, surface unread count more boldly.
24. **A signature moment**: one page (the dashboard?) gets distinctive treatment — a hero band with the school name, a "season summary" stat row, a Novi-themed empty state illustration. Make at least one page memorable.

## Closing assessment

**The functional rewrite is a success. The visual identity is not yet a product.** What's there is a faithful, structurally sound implementation of shadcn/ui defaults — the same template thousands of projects use. What's missing: a typographic voice, a brand color, motion language, and one moment of delight.

The fastest path to a real product feel: load a display font, adopt a Novi-green primary token, fix the eight CRITICAL bugs above, and pick one page (the dashboard) to make distinctive. Total effort: ~3-5 days of focused design work.
